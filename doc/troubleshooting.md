# 避坑指南与 Debug 日志 (Troubleshooting)

## 1. 渲染：基础世界不加载 (Invisible World)
*   **症状**：进入游戏只看到纯色背景或黑屏，按 ESC 退出后能看到相机在空中，但地面为空白。
*   **原因**：`WorldManager.update()` 被错误地包裹在 `if (controls.isLocked)` 判断内。由于世界未加载，`findSafeSpawn` 永远找不到地面，玩家一直处于 `hasSpawned = false` 的挂起状态。
*   **修复**：将世界更新逻辑移出锁定判断。即使在锁定前，也允许区块异步加载与渲染。

## 2. 交互：控制台状态回路与 UI 反馈
*   **症状**：按 T 键打开控制台后，输入字符会触发快捷栏切换；或者指令执行后玩家看不见任何反馈。
*   **修复**：
    1.  **输入拦截**：在全局 `keydown` 监听器中增加 `if (isConsoleOpen) return;`。
    2.  **状态路由**：引入 `isOpeningConsole` 标记位，确保 `unlock` 事件能正确导航到控制台 UI。
    3.  **内显日志**：在 `index.html` 中新增 `#console-log` 区域，并将 `CommandParser` 的返回结果通过 `addConsoleMsg` 实时打印在屏幕上，而非仅仅打印到开发者工具。

## 3. 存储：全域持久化与 Forced Air (ID 255) 丢失
*   **症状**：修改了视距外的方块没有保存；或者在地形生成前挖掉的方块重载后恢复。
*   **修复**：
    1.  **无视内存限制**：重构 `WorldManager.setBlock`，将 `saveChunkDelta` 彻底移出 `if (chunk)` 判断。
    2.  **强制空气保护**：在持久化阶段使用 `finalType`。当判定目标位置地形尚未就绪时，持久化 ID 为 255（强制空气），确保该标记能写入 IndexedDB 并成功拦截后续的 Worker 地形生成。

## 4. 渲染：贪婪网格化 (Greedy Meshing) 几何爆炸
*   **症状**：屏幕出现跨越空间的黑色拉伸三角形。
*   **修复**：当前已回滚至稳定版。教训：必须严格校验法线与 UV 数组的对齐长度。

## 5. 物理：出生点判定歧义
*   **原因**：`getHighestBlock` 在没找到地表时返回 0，与真实高度 0 产生冲突。
*   **修复**：将“未找到”状态显式修改为返回 **`null`**。

## 6. 交互：背包满载丢物品
*   **修复**：在挖掘和关闭 UI 两个关键节点增加 `addItem` 成功率校验，失败时强制拦截操作。

## 7. 物理：生物碰撞优化
*   **症状**：生物（如猪）会卡进墙角、台阶，或者在边缘穿模、悬浮。
*   **修复**：在 `Mob.js` 中实现了完整的 AABB 碰撞检测。现在生物会根据其半径和高度检查周围所有受影响的方块，解决了单点检测导致的各种物理 Bug。

## 8. 指令系统：非法数值校验
*   **症状**：`/give` 指令可以给予负数或零，导致背包数据异常。
*   **修复**：在 `CommandParser.js` 中增加了正数校验，并会根据 `addItem` 的布尔返回值给玩家提供准确的反馈。

## 11. 掉落物：磁力吸附无法拾取 (Logic Deadlock)
*   **症状**：掉落物会飞向玩家，但在玩家腰部反复横跳/抖动，无法进入背包。
*   **原因**：吸附的目标点（玩家中心/腰部）与距离判定点（玩家眼睛/相机）不一致。当物品飞到腰部时，距离眼睛的距离始终约为 1.0，无法满足 `< 0.5` 的拾取条件。
*   **修复**：在 `ItemDropManager.js` 中统一计算 `playerCenter` (相机位置向下偏移)，将吸附目标点与距离判定点合二为一。

## 12. Web Worker 通信：ArrayBuffer 转移导致数据丢失
*   **症状**：世界生成后走几步直接掉进虚空，物理碰撞彻底失效。
*   **原因**：在 `WorldManager` 向 Worker 发送网格化请求时，将存储体素数据的 `Uint8Array.buffer` 放入了 `transfer` 数组。这会导致主线程立刻丧失对该内存的所有权，数组长度变为 0（Detached 状态），从而毁掉了主线程的物理碰撞数据。
*   **修复**：向 Worker 传递只读数据时，依赖结构化克隆（Structured Clone）进行内存拷贝，不要使用 `transfer`。

## 13. 物理：掉落物撞墙“乘电梯”与穿透天花板
*   **症状**：掉落物碰到墙壁会无视重力向上爬；向上抛出时会穿透头顶的方块。
*   **原因**：未将垂直（Y轴）与水平（X/Z轴）的碰撞检测分离。撞墙时只要发生碰撞就把 Y 轴拉高；天花板检测时未判断运动方向。
*   **修复**：将物理检测拆分为三个轴的独立判定，并严格根据 `deltaPos.y` 的正负（上升/下落）来执行地面或天花板的碰撞拦截。

## 14. 渲染：区块对角线 AO 阴影残留
*   **症状**：破坏或放置区块角落的方块后，斜对角区块的阴影没有实时更新。
*   **原因**：`setBlock` 时仅仅 `markDirty` 了上下左右 4 个相邻区块，遗漏了对角线的 4 个区块，导致依赖对角线数据的 AO 阴影无法重绘。
*   **修复**：在修改区块边界方块时，同步判断并标记对角线区块的脏状态。
