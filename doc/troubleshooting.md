# 避坑指南与 Debug 日志 (Troubleshooting)

## 1. 渲染：基础世界不加载 (Invisible World)
*   **症状**：进入游戏只看到纯色背景或黑屏，按 ESC 退出后能看到相机在空中，但地面为空白。
*   **原因**：`WorldManager.update()` 被错误地包裹在 `if (controls.isLocked)` 判断内。由于世界未加载，`findSafeSpawn` 永远找不到地面，玩家一直处于 `hasSpawned = false` 的挂起状态。
*   **修复**：将世界更新逻辑移出锁定判断。即使在锁定前，也允许区块异步加载与渲染。

## 2. 交互：控制台状态回路与 UI 反馈
*   **症状**：按 T 键打开控制台后，输入字符会触发快捷栏切换；或者指令执行后玩家看不见任何反馈。
*   **修复**：
    1.  **输入拦截**：在全局 `keydown` 监听器中增加 `if (isConsoleOpen) return;`。
    2.  **状态路由**：引入 `isOpeningConsole` 标记位，确保 `unlock` 事件能正确导航到控制台 UI。
    3.  **内显日志**：在 `index.html` 中新增 `#console-log` 区域，并将 `CommandParser` 的返回结果通过 `addConsoleMsg` 实时打印在屏幕上，而非仅仅打印到开发者工具。

## 3. 存储：全域持久化与 Forced Air (ID 255) 丢失
*   **症状**：修改了视距外的方块没有保存；或者在地形生成前挖掉的方块重载后恢复。
*   **修复**：
    1.  **无视内存限制**：重构 `WorldManager.setBlock`，将 `saveChunkDelta` 彻底移出 `if (chunk)` 判断。
    2.  **强制空气保护**：在持久化阶段使用 `finalType`。当判定目标位置地形尚未就绪时，持久化 ID 为 255（强制空气），确保该标记能写入 IndexedDB 并成功拦截后续的 Worker 地形生成。

## 4. 渲染：贪婪网格化 (Greedy Meshing) 几何爆炸
*   **症状**：屏幕出现跨越空间的黑色拉伸三角形。
*   **修复**：当前已回滚至稳定版。教训：必须严格校验法线与 UV 数组的对齐长度。

## 5. 物理：出生点判定歧义
*   **原因**：`getHighestBlock` 在没找到地表时返回 0，与真实高度 0 产生冲突。
*   **修复**：将“未找到”状态显式修改为返回 **`null`**。

## 6. 交互：背包满载丢物品
*   **修复**：在挖掘和关闭 UI 两个关键节点增加 `addItem` 成功率校验，失败时强制拦截操作。

## 7. 物理：生物碰撞优化
*   **症状**：生物（如猪）会卡进墙角、台阶，或者在边缘穿模、悬浮。
*   **修复**：在 `Mob.js` 中实现了完整的 AABB 碰撞检测。现在生物会根据其半径和高度检查周围所有受影响的方块，解决了单点检测导致的各种物理 Bug。

## 8. 指令系统：非法数值校验
*   **症状**：`/give` 指令可以给予负数或零，导致背包数据异常。
*   **修复**：在 `CommandParser.js` 中增加了正数校验，并会根据 `addItem` 的布尔返回值给玩家提供准确的反馈。

## 11. 掉落物：磁力吸附无法拾取 (Logic Deadlock)
*   **症状**：掉落物会飞向玩家，但在玩家腰部反复横跳/抖动，无法进入背包。
*   **原因**：吸附的目标点（玩家中心/腰部）与距离判定点（玩家眼睛/相机）不一致。当物品飞到腰部时，距离眼睛的距离始终约为 1.0，无法满足 `< 0.5` 的拾取条件。
*   **修复**：在 `ItemDropManager.js` 中统一计算 `playerCenter` (相机位置向下偏移)，将吸附目标点与距离判定点合二为一。

## 12. Web Worker 通信：ArrayBuffer 转移导致数据丢失
*   **症状**：世界生成后走几步直接掉进虚空，物理碰撞彻底失效。
*   **原因**：在 `WorldManager` 向 Worker 发送网格化请求时，将存储体素数据的 `Uint8Array.buffer` 放入了 `transfer` 数组。这会导致主线程立刻丧失对该内存的所有权，数组长度变为 0（Detached 状态），从而毁掉了主线程的物理碰撞数据。
*   **修复**：向 Worker 传递只读数据时，依赖结构化克隆（Structured Clone）进行内存拷贝，不要使用 `transfer`。

## 13. 物理：掉落物撞墙“乘电梯”与穿透天花板
*   **症状**：掉落物碰到墙壁会无视重力向上爬；向上抛出时会穿透头顶的方块。
*   **原因**：未将垂直（Y轴）与水平（X/Z轴）的碰撞检测分离。撞墙时只要发生碰撞就把 Y 轴拉高；天花板检测时未判断运动方向。
*   **修复**：将物理检测拆分为三个轴的独立判定，并严格根据 `deltaPos.y` 的正负（上升/下落）来执行地面或天花板的碰撞拦截。

## 14. 渲染：区块对角线 AO 阴影残留
*   **症状**：破坏或放置区块角落的方块后，斜对角区块的阴影没有实时更新。
*   **原因**：`setBlock` 时仅仅 `markDirty` 了上下左右 4 个相邻区块，遗漏了对角线的 4 个区块，导致依赖对角线数据的 AO 阴影无法重绘。
*   **修复**：在修改区块边界方块时，同步判断并标记对角线区块的脏状态。

## 15. AI 寻路：无法跳跃或爬坡 (Vertical Pathfinding)
*   **症状**：僵尸只能在同一平面上移动，遇到一格高的台阶也无法跳上，寻路失效。
*   **原因**：A* 算法在判断垂直移动可行性时，将目标位置下方的方块误判为必须是实体方块。对于向上跳跃，下方是当前的空气。且 `getNeighbors` 缺少对角线与垂直方向的邻居节点。
*   **修复**：补全 26 邻居节点获取，并在 `isTraversable` 中放宽垂直移动条件，允许在脚下有实体支撑的情况下进行 `y+1` 跳跃。

## 16. 物理：掉落物反复上下抖动 (Physics Jitter)
*   **症状**：物品掉落在地面上后，不仅在视觉上悬浮，其整体物理坐标也会在 1 格高度内疯狂跳动。
*   **原因**：为了避免正弦波悬浮动画导致底部穿透方块，强制将碰撞着陆的物理坐标抬高。但这使得下一帧掉落物物理盒底部处于空气中，导致物理引擎判定可以继续下落。
*   **修复**：将物理碰撞包围盒 (Group) 紧贴地面，仅在视觉网格 (Mesh) 层面进行 `+ 0.1` 偏移并播放正弦波悬浮动画，实现物理与视觉解耦。

## 17. 交互：无法在水中挖掘方块 (Water Mining)
*   **症状**：玩家在水中或透过水面，长按左键挖掘下方的泥土/沙子毫无反应。
*   **原因**：`animate` 循环中判断挖掘进度射线检测时，未将 `ID=3` 的水方块过滤，导致选中的永远是水面，`hitSameBlock` 判断不断失败，进度清零。
*   **修复**：在循环更新射线时，添加对 `blockId !== 3` 的过滤逻辑，使其与 `mousedown` 开始挖掘时的判定保持一致。

## 18. 物理：方块放置挤压实体 (Entity Cramming)
*   **症状**：玩家可以向僵尸或猪的身体里直接放置方块，造成严重的穿模。
*   **原因**：右键放置判定仅检查了即将放置的方块 AABB 是否与玩家自身的 AABB 重叠，未遍历场景内的其他活动生物。
*   **修复**：引入对 `mobManager.mobs` 中所有存活生物 AABB 的遍历与重叠检测，如果存在交集则拦截放置行为。

## 19. 性能：Web Worker 内存泄漏与 SharedArrayBuffer
*   **症状**：游戏运行一段时间后或者高频加载区块时，浏览器内存占用飙升，Worker 线程假死导致地形不再生成。
*   **原因**：每次需要网格化时都给 Worker 发送一份完全 `clone` 的巨大 `Uint8Array`，或者错误使用了所有权转移（Transfer）导致数据脱离。后来引入 `SharedArrayBuffer` (SAB) 时，由于每次实例化新的 `VoxelWorld` 都在分配新的 SAB 却没有被垃圾回收。
*   **修复**：通过对象池或者复用机制处理 SAB，并正确配置 Vite 的 Cross-Origin 隔离头 (`Cross-Origin-Opener-Policy` 和 `Cross-Origin-Embedder-Policy`) 确保 SAB 可用，消除了零拷贝下的内存泄漏。

## 20. 性能：DDA 射线检测死锁
*   **症状**：当玩家直视正上方 (x=0, z=0) 挖掘时，页面直接卡死冻结。
*   **原因**：在实现 Fast Voxel Traversal (DDA) 算法时，若射线的方向向量某一轴分量为 0，会导致 `tDelta` 计算为 `Infinity`。在特定的精度退化下，`tMax` 可能变为 `NaN`，从而进入无限 `while` 循环。
*   **修复**：在 DDA 循环中增加了一个绝对的安全迭代计数器 (`maxIterations = 200`)，一旦达到阈值强制跳出循环，防止极端角度造成的浏览器进程锁死。

## 21. 交互：背包满时的 UI 软死锁
*   **症状**：当背包的 36 个槽位全部塞满，且玩家鼠标上或者 2x2 合成格内还有物品时，按 `E` 键或 `ESC` 键无法关闭背包，玩家被永远困在 UI 界面中。
*   **原因**：早期的逻辑是“如果物品无法退回背包，则拦截关闭事件并提示玩家”。但因为游戏缺乏主动将物品“丢出背包外”的机制，导致了死锁。
*   **修复**：重构关闭界面的退回逻辑：当尝试把鼠标和合成格的物品塞回背包失败后，不再阻挡关闭事件，而是直接调用 `itemDropManager.spawn` 将这些多余的物品以掉落物的形式生成在玩家脚下。

## 22. 存储：防抖持久化 (Debounced Persistence) 在页面关闭时丢失
*   **症状**：挖掉一个方块后立刻按 F5 刷新页面或者关闭标签页，重新进入游戏后发现方块又回来了。
*   **原因**：为了避免挖掘时疯狂占用 IndexedDB 导致主线程掉帧，引入了 `setTimeout` 的 500ms 写缓冲。但如果在这 500ms 内页面被卸载，`_flushPersistenceBuffer` 就永远不会执行。
*   **修复**：在 `WorldManager` 中监听了 `beforeunload` 与 `visibilitychange` 事件，并且使用同步的 `localStorage` 作为紧急回退存储，在下一次游戏加载时自动恢复未能写入 IndexedDB 的数据。
