# 避坑指南与 Debug 日志 (Troubleshooting)

## 1. 渲染：深度排序错乱（透视/空间颠倒）
* **症状**：远处的方块渲染在近处的方块之上，画面类似“纪念碑谷”般错乱。
* **原因与修复**：确认 Three.js 材质设置中 `depthTest: true` 和 `depthWrite: true`。确保非玻璃/水方块的 `transparent` 属性为 `false`。

## 2. 渲染：单面破洞/面片丢失
* **症状**：大部分渲染正常，但特定方向（如某个侧面）看过去能直接看到天空背景。
* **原因与修复**：
    1.  **顶点环绕顺序错误**：WebGL 会剔除顺时针排列的三角形。检查生成该面 `indices` 时，两个三角形的顶点连接顺序必须是**逆时针 (CCW)**。
    2.  **相邻检测方向写反**：检查 `isAir(x+dx, y+dy, z+dz)` 中的偏移量是否与当前生成的面法线方向完全一致（避免 Copy-Paste 错误）。

## 3. 物理：玩家落地后疯狂抽搐/反复降落
* **症状**：玩家触地后视野剧烈抖动，或者后台坐标一直穿模后被拉回。
* **原因与修复**：碰撞检测触发修正坐标后，**忘记清零垂直动量**。必须在碰撞修正代码中加入 `velocity.y = 0`，否则累积的重力会在下一帧瞬间把玩家重新砸入地下。

## 4. 物理：玩家掉入虚空无限循环 (Respawn Loop)
* **症状**：玩家穿过地面一直往下掉，触发安全高度后重置到天上，继续掉落穿模。
* **排查方向**：重点检查从世界真实坐标 (World Pos) 映射到 Chunk 数组 1D 索引 (Index) 的数学转换公式。如果公式有误，底层碰撞检测查到的永远是 0 (空气)。

## 5. 物理：出生卡墙（按 F 切换飞行才能移动）
* **症状**：刚进游戏时，玩家固定在原地无法走动和跳跃，但视角可以转。
* **原因与修复**：将玩家初始传送到最高方块上方时，忘记把脚部高度 (`y + 1`) 和眼部高度 (`eyeHeight`) 分开计算。玩家直接被置于最高方块上，其碰撞箱底部与实体方块重叠。修复方法：`camera.position.set(spawnX, y + 1 + eyeHeight, spawnZ);`。

## 6. 世界生成：树木悬空一层
* **症状**：所有的树都在离地一格的空中漂浮。
* **原因与修复**：在 3D 密度地形中，其实际地形高度（由 Density 决定）可能低于基础高度 `baseHeight`，直接在 `baseHeight + 1` 处生成树会导致悬空。必须通过从上往下扫描 `localData` 的实际方块数据找到真正的 `surfaceY` 并在 `surfaceY + 1` 处长树。

## 7. 架构漏洞：异步构建竞争与网格过期
* **症状**：玩家挖掘了方块或地形重建时，偶尔会“闪回”成旧的状态。
* **原因与修复**：`await getChunkDelta()` 异步获取期间，可能有新的构建请求发出，但旧请求之后拿到更新的 `version` 被 Worker 处理。修复方法：必须在 `await` 之前**第一时间分配并记录请求 ID**，在 `await` 结束后**校验 ID 是否仍匹配**，若不匹配说明过期，立刻终止。

## 8. 架构漏洞：随机性破坏地形增量持久化
* **症状**：每次重新进入同一坐标时，树木位置会变，可能覆盖掉玩家在树上建的房子。
* **原因与修复**：生成地形和植被时使用了 `Math.random()`。体素游戏的所有生成必须是**基于世界坐标确定性的 (Deterministic)**，必须使用固定的哈希伪随机数替换 `Math.random()`，保证多次生成结果完全一致。
