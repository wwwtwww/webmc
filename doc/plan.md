# Minecraft Clone 开发进度与计划

## 1. ✅ 已实现功能 (Completed)

### 渲染与核心系统
- [x] **基础框架**：基于 Vite + Three.js 的高效 ESM 开发环境。
- [x] **体素引擎**：使用 `Uint8Array` 优化内存，支持 16x16x16 区块存储。
- [x] **渲染优化**：
    - **面剔除 (Face Culling)**：仅渲染暴露在空气中的面。
    - **棋盘格变色**：基于坐标奇偶性的顶点变色，清晰分辨方块边界。
    - **逆时针绕序 (CCW)**：修复背面剔除引起的透视问题。
- [x] **视觉增强 (Fog)**：实现线性雾效果，完美掩盖远处区块的动态加载边界。

### 物理与交互
- [x] **第一人称控制**：`PointerLockControls` 实现鼠标视角与 WASD 移动。
- [x] **物理碰撞**：实现 AABB 碰撞检测与重力系统，支持滑动碰撞与跳跃。
- [x] **方块交互**：实现射线检测（Raycaster），支持左键挖掘和右键放置。

### 世界生成
- [x] **无限世界**：`WorldManager` 自动根据玩家位置加载/销毁区块（视距=3）。
- [x] **无缝地形**：使用确定性 Simplex Noise 及全局坐标采样，确保跨区块山脉连绵。
- [x] **程序化植被**：实现基于确定性哈希的树木（树干+树叶）自动生成。
- [x] **内存管理**：严谨的区块销毁逻辑，确保 GPU 显存（Geometry/Material）正确释放。

## 3. 下一步建议
建议接下来的开发重点放在 **多线程计算 (Web Workers)**。随着渲染距离的增加或地形复杂度的提升，主线程生成网格会导致明显的掉帧。将计算移至 Worker 可以保证极致的丝滑体验。
