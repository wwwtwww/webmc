# WebMC 项目开发计划书

## 1. 已完成项 (Completed)

### 核心架构
- [x] **现代工程环境**：基于 Vite + Three.js 搭建高性能 ESM 开发工作流。
- [x] **文档体系建设 (NEW)**：根据最新版本更新 README.md 核心特性说明与控制台使用指南。
- [x] **超高区块生成**：支持 16x256x16 规格区块，实现纵向 256 格的完整世界建模。
- [x] **多线程加速**：地形生成与 Mesh 网格化逻辑完全移至 Web Worker，主线程零阻塞。
- [x] **环境光遮蔽 (AO)**：实现基于顶点亮度的实时计算 AO 系统，增强体素世界的层次感。
- [x] **动态环境系统**：具备实时的昼夜交替逻辑，天空盒颜色、光照与雾效随时间自然过渡。
- [x] **异步对齐优化**：通过 Request ID 机制解决了主线程与 Worker 通信中的异步状态对齐问题。

### 玩家控制与交互
- [x] **物理引擎**：实现严密的 AABB 碰撞检测、重力感应及丝滑的移动算法。
- [x] **开发者控制台 (NEW)**：支持 `T` 或 `键唤出，集成 `/time`, `/give`, `/tp`, `/clear-mobs` 指令。
- [x] **飞行模式 (NEW)**：支持 `F` 键一键切换上帝模式 (God Mode)，支持穿墙与自由飞行。
- [x] **调试仪表盘 (F3)**：实时显示 FPS、当前坐标、生物群落信息及活动实体数量。
- [x] **交互距离限制**：通过 Raycaster 算法精准控制 5 格以内的方块挖掘与放置。
- [x] **安全出生逻辑**：加入 Safe Spawn 检测机制，确保玩家始终出生在露天安全地表。

### 智能生物系统 (Mob AI)
- [x] **生物管理器**：`MobManager` 统一管理所有实体（Mob）的生命周期、渲染与状态。
- [x] **寻路算法 (NEW)**：集成具备垂直跳跃避障能力的 A* 寻路，支持僵尸在 3D 空间的路径搜索。
- [x] **有限状态机 (FSM)**：实现生物在漫游、追踪、受击、死亡状态间的自动切换逻辑。
- [x] **受击反馈系统**：包含完整的伤害判定、击退效果、红闪提示及死亡消失动画。
- [x] **死亡掉落机制 (NEW)**：生物死亡后会生成掉落物，并具备磁力吸附自动拾取功能。

### 背包、UI 与合成
- [x] **多格背包系统**：实现 36 槽位背包管理及 1-8 快捷栏同步切换。
- [x] **2D 物品渲染**：独立 UI 渲染器实现 2D 物品图标叠加，支持透明度与缩放渲染。
- [x] **2x2 随身合成**：支持在背包界面进行基础物品的实时合成公式识别与预览。
- [x] **交互优化 (Bug Fixes)**：累计修复超过 35 个涉及物品槽位、AI 路径死锁及渲染崩溃的 Bug。

### 地形与存储
- [x] **程序化地形**：基于 3D Simplex Noise 生成平滑过渡的沙漠、草地与雪山等生物群落。
- [x] **植被生态系统**：支持程序化橡树、灌木及地表杂草的随机分布生成。
- [x] **持久化存储**：结合 `Dexie.js` 将玩家对世界的 Deltas 修改实时持久化至 IndexedDB。
- [x] **音效管理 (AudioManager)**：实现基于 LRU 策略的音频资源管理，支持 3D 空间音效播放。

## 2. 待办事项 (To-Do List)

### 性能极致优化 (短期)
- [ ] **射线检测重构**：优化 `animate` 循环，采用区块边界 DDA 算法替代常规射线，降低 CPU 开销。
- [x] **WebGL 属性缓冲区优化 (Bug 45)**：在 `Chunk._updateMeshGeometry` 中实现 BufferAttribute 复用，使用 `.set()` 更新数据而非频繁销毁重建，解决 GPU 阻塞与显存碎片问题。
- [ ] **内存碎片整理**：引入 Chunk Buffer 池化技术，使用 `setDrawRange` 优化显存数据交换。
- [ ] **寻路算法调优**：在 `Pathfinder.js` 中引入 `BinaryHeap` 二分堆，将寻路开销从 O(N) 降至 O(log N)。
- [ ] **Greedy Meshing**：重构网格合并算法，通过合并同类表面减少 60% 以上的渲染顶点数量。
- [ ] **LOD 视距支持**：开发远景低精细度渲染模式，支持 12 个区块以上的超远视距。

### 生存模式扩展 (中期)

- [ ] **呼吸与潜水**：加入入水后的氧气条 UI，实现窒息伤害与气泡反馈动画。
- [ ] **物品耐久度**：实现工具（镐、铲）的使用次数限制及损坏逻辑。

### 联机与工程 (长期)
- [ ] **多人联机模式**：基于 WebSocket 或 WebRTC 实现玩家坐标、动画及方块修改的实时同步。
- [ ] **自动化测试**：进一步提高 `Vitest` 覆盖率，尤其是针对核心物理边界与 Worker 通信的测试。

## 3. 设计原则
- **3x3 开发规范**：每个功能模块需满足模块化、高性能、可测试。
- **UI/Logic 分离**：确保业务逻辑与 Three.js 渲染层解耦，便于维护。
- **内存安全**：严格监控 Web Worker 传递过程中的对象所有权，避免内存泄露。
