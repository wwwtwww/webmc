# WebMC 项目计划书

## 1. 已完成事项 (Completed)

### 基础架构
- [x] **工程化环境**：基于 Vite + Three.js 构建 ESM 模块化开发环境
- [x] **项目文档 (NEW)**：补全架构说明 README.md、开发计划与路线图
- [x] **区块系统**：支持 16x256x16 的区块划分，并实现 256 层高度限制
- [x] **异步网格化**：将 Mesh 生成逻辑完全移至 Web Worker，实现零阻塞渲染
- [x] **环境遮挡 (AO)**：实现基于邻居方块状态的逐顶点 AO 阴影平滑渲染
- [x] **动态更新**：支持方块挖掘与放置时的局部网格实时动态刷新
- [x] **请求 ID 校验**：引入 Request ID 机制解决异步 Worker 回传数据的版本冲突

### 核心交互
- [x] **物理引擎**：实现 AABB 碰撞检测、重力感应与摩擦力物理逻辑
- [x] **控制台系统 (NEW)**：按 `T` 唤起控制台，支持 `/time`, `/give`, `/tp`, `/clear-mobs` 等指令
- [x] **飞行模式 (NEW)**：支持 `F` 键切换上帝模式（飞行/穿墙状态）
- [x] **调试面板 (F3)**：实时显示 FPS、坐标信息、区块状态及内存监控
- [x] **射线交互**：基于 Raycaster 实现 5 格范围内的方块精准拾取与交互
- [x] **安全出生点**：实现 Safe Spawn 逻辑，防止玩家掉入虚空或卡入实体

### 生物系统 (Mob AI)
- [x] **生物管理器**：`MobManager` 统一管理所有 Mob 实例的生命周期与更新
- [x] **寻路算法 (NEW)**：基于 A* 算法实现支持跳跃与避障的 3D 空间寻路
- [x] **状态机 (FSM)**：实现生物在闲逛、追逐、受击、死亡状态间的自动切换
- [x] **碰撞处理**：完善生物与地形、生物与玩家间的物理碰撞体积校验
- [x] **伤害系统 (NEW)**：支持玩家攻击反馈、受击变红效果及死亡物品掉落

### 背包与 UI 系统
- [x] **快捷栏系统**：支持 36 格背包空间，支持 1-8 数字键快速切换槽位
- [x] **2D 物品渲染**：实现 UI 层面的 2D 贴图与堆叠数量的 Canvas 渲染
- [x] **2x2 合成**：支持背包内的基础 2x2 合成逻辑与动态配方匹配
- [x] **缺陷修复 (Bug Fixes)**：修复了背包第 35 格数据残留及 AI 目标丢失等多个缺陷

### 系统底层
- [x] **地形生成**：基于 3D Simplex Noise 实现多生物群落的平滑地形生成
- [x] **昼夜系统**：实现随时间平滑过渡的天空盒颜色与全局光照强度变化
- [x] **数据持久化**：集成 `Dexie.js` (IndexedDB) 存储玩家对世界的增量修改数据
- [x] **音频系统 (AudioManager)**：实现基于 LRU 缓存的 3D 空间音效管理

### 性能专项 (Optimization)
- [x] **存档持久化写缓冲 (Bug 48)**：在 `WorldManager.js` 中引入写缓冲防抖机制，合并高频 I/O 请求，解决挖掘时的性能卡顿。
- [x] **内存 GC 压力优化 (Bug 47)**：在 `main.js`, `Mob.js`, `ItemDrop.js` 中使用预分配的全局临时变量复用，消除 Vector3 实例化的 GC 开销。
- [x] **DDA 射线检测重构 (Bug 46 & 49)**：引入 DDA 步进算法取代原生射线检测，解决 O(N) 级帧率衰减，效率提升 10-50 倍。
- [x] **WebGL 显存优化 (Bug 45)**：在 `Chunk._updateMeshGeometry` 中通过 `BufferAttribute.set()` 复用缓冲区并配合 `setDrawRange` 渲染，避免 GPU 阻塞。

## 3. 架构建议
- **3x3 区块刷新**：挖掘方块时仅更新受影响的 $3 \times 3$ 区块集，而非全量刷新
- **UI/逻辑分离**：进一步解耦 Three.js 渲染对象与背包数据层的逻辑状态
- **任务分片**：将大型 Web Worker 计算任务进行时间片切割，防止 Worker 线程阻塞
