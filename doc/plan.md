# Minecraft Clone 开发进度与计划

## 1. ✅ 已实现功能 (Completed)

### 渲染与核心系统
- [x] **基础框架**：基于 Vite + Three.js 的高效 ESM 开发环境。
- [x] **高度扩展 (16x256x16)**：支持 256 层垂直空间，彻底解决宏观地形限制。
- [x] **多线程网格化**：Web Worker 处理环境数据，采用基础表面剔除 (Naive Surface Culling) 算法。
- [x] **双通道渲染**：不透明与透明（水、玻璃、树叶）Mesh 独立渲染，支持环境光遮蔽 (AO)。
- [x] **天空与环境系统 (NEW)**：引入 `SkyManager` 统一驱动场景颜色、雾气、环境光强度及太阳运动轨迹。
- [x] **渲染安全性**：全局 Request ID 校验与 ID 255 "强制空气" 机制，解决异步加载导致的渲染闪烁与脏数据问题。

### 存储与持久化
- [x] **增量持久化存储**：基于 `Dexie.js` (IndexedDB) 实现。仅存储增量 (Deltas)，并在 Worker 线程合并生成地形。
- [x] **全域持久化**：`setBlock` 现在支持对非内存区块的实时保存，确保所有修改均不会丢失。

### 物理与交互
- [x] **双模式控制器**：生存模式 (AABB 碰撞) 与 飞行模式 (Noclip)。
- [x] **开发者控制台 (NEW)**：支持 `T` 键呼出指令行，支持 `/time`, `/give`, `/tp` 等调试指令。
- [x] **交互系统**：支持 1-8 数字键切换方块，支持 3D 图标化的 UI 快捷栏。
- [x] **调试增强 (F3/HUD)**：实时坐标、FPS、区块坐标、生物群落显示。
- [x] **真实挖掘机制**：长按左键持续挖掘，配套 UI 进度条 (空手 4 秒挖完)。限制 5 格最大触及距离 (MAX_REACH)。
- [x] **安全拾取循环**：挖掘方块后方块进入背包。具备满背包保护（拦截方块破坏）。
- [x] **安全出生点 (Safe Spawn)**：已修复高度判定歧义与坐标中心化问题，确保玩家平稳出生。

### 背包与合成
- [x] **背包系统 (Inventory)**：实现 36 槽位数据模型，支持点击拿起/点击放下的交互模式。
- [x] **安全回收**：关闭背包时自动尝试放回手持物品，若背包已满则拦截关闭操作，彻底防止物品丢失。
- [x] **合成系统 (Crafting)**：支持背包内的 2x2 合成。

### 世界生成
- [x] **多重生物群落**：通过 2D 群落噪声实现沙漠 (Desert)、雪山 (Snowy) 与草地 (Grass) 的平滑过渡。已统一主线程与 Worker 的判定口径。
- [x] **确定性生成**：引入固定种子的随机算法，确保同一世界坐标在不同会话中生成一致。
- [x] **程序化植被**：基于坐标确定性伪随机算法生成树木，通过垂直扫描修复了“浮空树”Bug。

### 音效系统
- [x] **AudioManager**：基于对象池与 LRU 机制实现音轨复用，支持挖掘、放置与动态音调偏移的脚步声。

## 2. 🚧 待办任务 (To-Do List)

### 第一优先级：性能与自动化
- [ ] **自动昼夜流逝**：在 `SkyManager` 中接入 `deltaTime` 驱动，实现 20 分钟一个周期的自动光影变化。
- [ ] **贪婪网格化 (Greedy Meshing)**：合并相邻同材质面片以极大减少顶点数量（待解决属性对齐与绕序问题）。
- [ ] **视距裁剪优化**：视锥剔除 (Frustum Culling) 进一步减少 GPU 负担。

### 第二优先级：环境细节与生物
- [ ] **简单生物 AI**：实现基础的被动生物在地面随机移动。
- [ ] **云层系统**：高度确定的体积云或简单的面片云。
