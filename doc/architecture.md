# 架构设计 (Architecture)

## 1. 技术栈
* **核心环境**：Node.js
* **构建工具**：Vite (提供极速的热更新和现代前端工程化)
* **3D 渲染引擎**：Three.js
* **数据存储**：Dexie.js (IndexedDB 封装)，用于实现世界状态持久化。

## 2. 数据结构设计
* **Chunk (区块)**：核心单元为 16 x 256 x 16 的体素阵列。
* **存储策略**：
    * **主内存**：使用 `Uint8Array` 存储方块 ID。
    * **持久化**：采用 **Delta (增量) 存储模式**。仅记录玩家对世界进行的 `setBlock` 操作，数据库存储格式为 `{ chunkKey: "x,z", deltas: { "x_y_z": blockID } }`。

## 3. 异步并发管线
为了保证 60FPS 的流畅交互，系统将计算任务卸载至 Web Worker：
1. **渲染更新**：当区块可见或发生变化时，主线程向 Worker 发送环境数据。
2. **地形合并**：Worker 在地形生成算法产出的数据基础上，自动合并来自 Dexie 的历史修改 Deltas。
3. **网格化 (Meshing)**：Worker 执行 AO 顶点计算、面剔除并生成 `BufferGeometry` 数据。
4. **校验机制**：使用全局单调递增的 `version` ID。主线程仅接受版本号匹配的几何体数据，避免异步加载导致的“旧数据闪现”问题。

## 4. 关键子系统架构
* **音效系统 (AudioManager)**：
  - **对象池 (Object Pool)**：预加载固定数量的 `THREE.Audio` 实例，避免重复创建开销。
  - **LRU 轮转**：当播放请求超出池容量时，强制中断并重用最早播放的实例，保证声音叠加的流畅感。
  - **动态微调**：内置 `setDetune` 的随机偏移，减轻同一音效重复播放带来的机械感。
* **物理引擎 (Physics Engine)**：
  - **混合模式**：支持 Noclip 飞行与基于 AABB 的地表碰撞。
  - **重力状态机**：通过 `isReady` 标志位确保只有在玩家脚下地形加载完成后才启用重力，防止掉入未生成的虚空。
* **世界生成 (World Generation)**：
  - **3D Simplex Noise**：支持悬崖、洞穴与复杂的垂直结构。
  - **Smoothstep 平滑出生区**：在原点周围 30 格强制生成平原，并通过 S 曲线算法实现与噪声地形的自然过渡。
