# Player HUD Design

## 概述
为体素引擎添加一个常驻在屏幕左上角的玩家状态栏 (Player HUD)，使用原生 JavaScript 动态生成纯 HTML/CSS 结构，以维持体素游戏硬朗的视觉风格。

## 架构与组件
- **新模块**: `PlayerHUD.js`，独立负责 HUD 的创建和更新逻辑。
- **导出函数**: `initHUD()`，用于在游戏初始化时动态生成 DOM 节点并挂载到 `document.body` 或 `#app`。
- **导出函数 (预留)**: `updateHUDHealth(hp, maxHp)`，用于后续同步真实血量。

## 视觉设计
- **定位**: 绝对定位（`position: absolute; top: 20px; left: 20px;`），确保不遮挡核心视野。
- **容器样式**: 采用半透明深色背景（`background-color: rgba(0, 0, 0, 0.5)`），内边距适中，圆角极小或无圆角以符合方块世界风格。
- **玩家名称**: 文本“YILE”，字体较大，颜色纯白，通过硬朗的黑色文字阴影（`text-shadow: 2px 2px 0 #000`）增加层次感。
- **血条槽**: 位于名字下方，外层是一个占位槽（如深灰色背景），内层为绿色的实际血条（默认宽度 `100%`），过渡平滑。

## 数据流与整合
- **引入点**: 将在 `main.js` 的初始化阶段导入并调用 `initHUD()`。
- **原有 UI 替换**: 目前的 `main.js` 中存在一个位于屏幕底部的简易 `#hp-ui`，未来可以考虑将其隐藏或移除，统一由 `PlayerHUD` 接管血量展示。

## 错误处理与兼容性
- **层级管理**: 设定足够高的 `z-index`，确保不会被渲染的 canvas 遮挡。
- **指针事件**: 设置 `pointer-events: none`，防止玩家在旋转视角时意外点击到 UI 导致鼠标焦点脱离。
