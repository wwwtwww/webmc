# WebMC 项目进度

## 1. 已完成事项 (Completed)

### 核心引擎
- [x] **模块化架构**：采用 Vite + Three.js 构建 ESM 插件化开发体系
- [x] **文档完善 (2024-04-30 UPDATED)**：完成了 89 项 Bug 修复的同步记录，更新了架构说明与开发规约
- [x] **高度限制**：支持 16x256x16 的垂直区块，适应 256 格世界高度
- [x] **区块网格化**：基于 Mesh 贪婪取样算法，支持 Web Worker 异步生成
- [x] **环境光遮蔽 (AO)**：实现方块边角阴影，并在区块边界与对角线处实现了平滑刷新
- [x] **视锥剔除**：根据相机朝向动态卸载不可见区块网格
- [x] **请求 ID 管理**：建立 Request ID 追踪机制，解决 Worker 消息乱序问题

### 玩家交互与 UI
- [x] **Player HUD (NEW)**：实现了基于 HTML/CSS 的原生 UI 模块，支持动态血条同步、准星及快捷栏反馈
- [x] **工具系统 (NEW)**：引入镐、斧、锹等工具，支持根据方块类型自动匹配最佳工具提速（最高 5 倍挖掘效率）
- [x] **3x3 合成系统 (NEW)**：完成交互式工作台界面，支持复杂的物品合成公式与产出堆叠处理
- [x] **物理碰撞加固**：彻底修复了掉落物抖动、穿墙、玩家窒息判定及坠落伤害逻辑
- [x] **命令系统**：按 `T` 唤起控制台，支持 `/time`, `/give`, `/tp`, `/clear-mobs` 等常用指令
- [x] **调试面板 (F3)**：实时显示真实 FPS、坐标、区块信息与内存占用

### 生物系统 (Mob AI)
- [x] **路径追踪 (A* Upgrade)**：实现支持垂直移动、跳跃判定与 26 方向探索的 A* 寻路算法，优化了 O(log N) 的搜索性能
- [x] **生物排斥与碰撞**：引入生物间水平排斥力，防止实体完全重叠（Entity Cramming）
- [x] **状态机 (FSM)**：生物具备“空闲”、“游荡”、“追逐”多种行为状态
- [x] **伤害反馈**：实现受击变红、击退物理反馈及 1 秒统一无敌帧（I-Frame）逻辑
- [x] **行为增强**：僵尸支持白天自燃、触墙立即跳跃及基于转向角度的速度缩放

### 持久化与性能优化
- [x] **89 项 Bug 修复总结**：全面清理了从逻辑漏洞、内存泄漏到 UI 冲突的所有已知问题
- [x] **SharedArrayBuffer 零拷贝**：主线程与 Worker 间通过 SAB 实现体素数据零拷贝传输，消除内存克隆开销
- [x] **DDA 射线检测**：引入体素步进算法（DDA），相比 Raycaster 提升方块定位效率 10-50 倍
- [x] **WebGL 缓冲区复用**：实现 `BufferAttribute` 池化，避免频繁申请 GPU 显存，显著降低 GC 压力
- [x] **批量持久化优化**：引入 500ms 写缓冲防抖，通过 Dexie.js 事务合并离散写入，并建立 `localStorage` 崩溃回退机制
- [x] **对象池技术**：在主循环中全面复用 Vector3 等对象，减少高频实例化导致的帧率波动

## 3. 核心限制说明
- **3x3 渲染限制**：目前受限于单线程 JS 性能，默认最大渲染范围维持在 3x3 个区块，开启 LOD 后可扩展
- **UI 交互冲突**：Three.js 鼠标锁定模式已与 DOM UI 完成解耦，确保打字时不触发游戏指令
- **Worker 约束**：所有 Web Worker 严禁操作 DOM，通过 SharedArrayBuffer 维持高性能状态同步
