# Tool Crafting System Design

## 概述
引入核心的工具合成系统（镐子、斧头等），以“永久工具 (无耐久度)”和“简易固定减免 (Simple Reduction)”的机制，为接下来的地下矿物探索打下基础。玩家需要通过 2x2 或 3x3 的合成界面打造工具，不同的工具会显著影响不同方块的挖掘速度，并决定是否能够获得对应的掉落物。

## 核心机制 (Mechanics)

### 1. 简易挖掘速度减免 (Simple Reduction)
*   **基础逻辑**：我们不再采用原版极其复杂的“硬度×工具乘数”计算，而是采用固定秒数机制。
*   **基础挖掘时间**：例如所有方块默认徒手挖掘需要 `1.5` 秒。
*   **工具减免**：当玩家手持合适的工具时，挖掘时间直接缩短为一个固定的快值（例如 `0.3` 秒）。
*   **工具类型映射**：
    *   **镐 (Pickaxe)**：针对石头、矿石等硬质方块。
    *   **斧 (Axe)**：针对木头、木板等木质方块。
    *   **锹 (Shovel)**：针对泥土、沙子、积雪等松软方块。

### 2. 挖掘等级锁 (Harvest Level)
*   **机制**：某些方块（如未来的石头、铁矿）具备 `minHarvestLevel` 属性。
*   **行为**：如果玩家手持的工具等级低于该要求（或者徒手），虽然依然可以（缓慢地）破坏该方块，但**不会产生任何掉落物**。

### 3. 工具数据结构 (Tool Data Structure)
在 `main.js` 或拆分的物品配置表中，为工具类物品增加特定属性：
```javascript
{
  id: 20, 
  name: '木镐', 
  color: '#8b5a2b', 
  isTool: true,
  toolType: 'pickaxe',
  toolLevel: 1 // 木=1, 石=2, 铁=3...
}
```
并在被挖掘的方块属性中标记其偏好的工具：
```javascript
{ 
  id: 15, 
  name: '石头', 
  preferredTool: 'pickaxe', 
  minHarvestLevel: 1 // 至少需要木镐(Level 1)才能掉落
}
```

## 合成系统扩展 (Crafting Expansion)

### 1. 2x2 随身合成 vs 3x3 工作台合成
*   **当前状态**：已经存在 2x2 的随身合成槽 (`CraftingManager.js`)。
*   **升级方案**：
    *   引入 3x3 的工作台界面状态。
    *   在 `index.html` 的 UI 中，通过 CSS Grid 动态切换 2x2 和 3x3 布局。
    *   当玩家右键点击世界中的“工作台”(ID: 11) 方块时，打开带有 3x3 网格的 UI，并将 `inventoryUI.isWorkbench` 标记为 `true`。
    *   `CraftingManager.checkRecipe` 需要支持 3x3 的输入矩阵解析。

### 2. 工具配方示例
*   **木镐 (Wooden Pickaxe)**
    *   材料：3x 木板(9)，2x 木棍(10)
    *   形状：顶部一排木板，中间一列木棍。必须在 3x3 工作台中合成。

## 错误处理与兼容性
*   **UI 切换**：关闭背包时，必须清理 3x3 槽位，将未使用的材料退回背包或丢弃在地上，并重置为 2x2 随身模式。
*   **射线检测**：右键工作台打开 UI 时，需要正确拦截方块放置逻辑（即右键工作台不应把手中的方块放在工作台表面，而是触发交互）。
